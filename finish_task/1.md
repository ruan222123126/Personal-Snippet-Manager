这份架构报告非常专业，逻辑清晰且技术选型（Next.js App Router + SQLite FTS5 + Shiki）非常有品位。既然你不想看代码，只想要**行动指南**，那我们按照**“数据驱动开发（Database-Driven Development）”**的思路来开启第一步。

第一步的核心目标是：**搭建骨架（Skeleton）与 铺设数据脊椎（Schema）。**

请按以下顺序执行操作，完成后把成果展示给我：

### 第一阶段：项目初始化与数据层构建

#### 1. 脚手架初始化 (Scaffolding)

* **动作**：初始化一个 Next.js 项目。
* **关键配置要求**：
* 必须启用 **TypeScript**（类型安全是本项目的核心）。
* 必须启用 **Tailwind CSS**（后续配合 Masonry 布局）。
* 必须启用 **App Router**（架构报告中明确依赖 RSC）。
* 配置好路径别名（`@/*`），方便后续引用 `lib` 和 `components`。


* **清理**：删掉默认生成的 `page.tsx` 中的样板代码，只留一个干净的 `<main>`。

#### 2. Prisma 环境搭建

* **动作**：安装 Prisma CLI 和 Client，初始化 SQLite 数据库。
* **Schema 定义**：根据你报告中的 **3.1 章节**，在 `prisma/schema.prisma` 中编写模型。
* 定义 `Snippet` 模型。
* 定义 `Tag` 模型。
* **重点**：定义显式的 `TagOnSnippet` 中间表（不要用隐式多对多），确保包含 `assignedAt` 字段。



#### 3. 实现 SQLite FTS5 (高阶步骤)

* **动作**：不要直接 `prisma db push`，我们需要通过 Migration 来定制 SQL。
* **执行流程**：
1. 执行 Prisma 命令创建一个 **Empty Migration**（空迁移文件）。
2. 找到生成的 `.sql` 文件。
3. **手动填入**：将报告 **3.2.1 章节** 中的原生 SQL 代码（创建虚拟表 `SnippetFTS` 和 3 个触发器 Trigger）复制进去。
4. 执行 Migrate Deploy 命令应用更改。
*此步骤至关重要，它决定了你后续的搜索功能是否是从一开始就是高性能的。*



#### 4. 建立核心库目录结构

* **动作**：虽然先不写代码，但要把文件位置占好，确立架构规范。在项目根目录下创建以下文件夹和空文件：
* `lib/prisma.ts` —— (用于放全局 Prisma Client 单例)
* `lib/shiki.ts` —— (用于放报告 4.1 提到的 Highlighter 单例)
* `components/ui/` —— (用于存放 UI 组件)
* `app/api/snippets/route.ts` —— (预留 API 路由位置)

当你完成上述步骤后，你应该达到以下状态：
