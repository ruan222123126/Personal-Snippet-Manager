### 5. 任务管理工作流程

#### 任务文件夹结构
```
[项目根目录]/
├── task/              # 待处理任务 - 从这里读取任务文件
├── finish_task/       # 已完成任务 - 完成后将任务文件移到这里
└── report/            # 完成报告 - 完成任务后在这里写报告
```

#### 处理任务时
1. 从 `task/` 读取任务文件
2. 按照开发规则完成任务
3. 将任务文件移到 `finish_task/`
4. 在 `report/` 写完成报告

**任务完成检查清单**:
- [ ] 任务需求完全实现
- [ ] 代码遵循项目规范
- [ ] 类型检查通过
- [ ] 任务文件已移到 `finish_task/`
- [ ] 完成报告已写在 `report/`
- [ ] 已发送桌面通知

### 6. 任务完成通知

完成任何任务时，使用以下命令发送桌面通知：

```bash
# 成功通知（绿色）
/path/to/project/.claude/task-notify.sh "✅ 任务完成" "具体任务描述" "normal"

# 警告通知（黄色）
/path/to/project/.claude/task-notify.sh "⚠️ 注意" "警告信息" "normal"

# 错误通知（红色）
/path/to/project/.claude/task-notify.sh "❌ 错误" "错误信息" "critical"
```

**支持的图标**:
- ✅ 成功/完成
- ⚠️ 警告/注意
- ❌ 错误/失败
- ℹ️ 信息/提示
- 🚀 新功能/发布
- 🐛 Bug 修复
- 🔧 配置/工具
- 📝 文档/注释
